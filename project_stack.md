# ОБЩАЯ ИНФОРМАЦИЯ

**Тема проекта:** Recipe Tree Visualizer — веб-сервис для извлечения рецептов крафта из модов Minecraft Java (.jar) и построения интерактивного нодового дерева зависимостей крафта

**Формат работы:** В команде (2 человека)

## Состав команды и роли

- **Колупаев Ефим:** Тимлид / Frontend-разработчик (визуализация дерева, React Flow) + интеграция с API
- **Бердюгин Евгений:** Архитектор + Backend-разработчик (парсер .jar, граф зависимостей, REST API) + DevOps (CI/CD, Docker, uv)

# ВЫБРАННЫЙ СТЕК ТЕХНОЛОГИЙ

## Язык программирования

- **Python 3.13** (backend)
- **TypeScript 5.x** (frontend)

## Основной фреймворк/библиотеки

### Backend

- **FastAPI + Uvicorn** — асинхронный API, авто-документация OpenAPI/Swagger
- **Pydantic v2** — валидация и схемы данных рецептов
- **zipfile (stdlib)** — чтение .jar (это обычные ZIP-архивы)
- **orjson** — быстрый парсинг тысяч JSON-файлов рецептов
- **networkx** — построение DAG, топосортировка, поиск циклов

### Frontend

- **React 19 + Vite** — SPA-каркас и сборка
- **@xyflow/react (React Flow v12)** — нодовый редактор/дерево
- **elkjs** или **@dagrejs/dagre** — автоматическая раскладка дерева
- **TanStack Query** — запросы к API и кэш

## Общие инструменты

- **uv** — менеджер пакетов/окружений (быстрая замена pip+venv+poetry)
- **pytest** — тестирование backend
- **Docker** — контейнеризация и единое окружение

## Система логирования

- **Loguru** (backend)

## Линтер (статический анализ)

- **Ruff** (lint + format, заменяет Flake8/Black/isort)
- **mypy** (проверка типов)
